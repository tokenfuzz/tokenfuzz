# Firefox subsystem overlay — the single source of truth for two things:
#
#   1. Path → focus-area attribution: longest-prefix match so a path like
#      dom/media/webcodecs/Foo.cpp resolves to "dom/media/webcodecs"
#      (lib/parse_state.py).
#   2. Cold-start subsystem candidates per agent mode: which subsystems a
#      browser-mode vs shell-mode agent may pick (bin/audit
#      browser_mode_subsystems / shell_mode_subsystems).
#
# Format: "<mode> <path>", one entry per line. mode is one of:
#   browser — reachable from web content via the browser launch path
#             (DOM, layout, editor, image decoders, fonts, parser/html).
#   shell   — reachable from the js shell / xpcshell (JS engine, JIT,
#             Wasm, irregexp, self-hosted builtins, xpcshell XPCOM).
#   both    — a valid cold-start candidate in either mode.
#
# Attribution uses every path regardless of mode; the mode column only
# gates which subsystems an agent may pick at cold start. Add a subsystem
# by adding a line — no code change. Comments (#) and blank lines ignored.
